```
Крок 1: Відкрий папку ai-ads-agent/ в Claude Code
```

## Всі 12 команд

### Моніторинг та звіти

#### `/dashboard` — Дашборд

**Для чого:** Побачити все одним поглядом. Всі акаунти, основні метрики.

**Як використовувати:**
```
/dashboard                           ← Всі акаунти, вчора
/dashboard today                     ← За сьогодні
/dashboard 7d                        ← За тиждень
"Покажи дашборд по Bas Dent за 7 днів"  ← Конкретний акаунт
```

**Що побачиш:**
- Таблиця: Акаунт → Витрати → Ліди → CPL → % від плану
- Для кожного акаунту: кампанії з метриками
- За запитом: деталі до рівня оголошень

**Рівні деталізації:**
- `account` — тільки загальне по акаунтах
- `campaign` — розбивка по кампаніях (за замовчуванням)
- `adset` — розбивка по групах оголошень
- `ad` — кожне окреме оголошення

---

#### `/ads-reporter` — Детальні звіти

**Для чого:** Глибокий аналіз з трендами, порівняннями, розбивкою по аудиторіях.

**Як використовувати:**
```
"Звіт по Bas Dent за сьогодні"
"Порівняй цей тиждень з минулим по Smile Clinic"
"Покажи звіт за 14 днів з розбивкою по віку"
"Тижневий звіт по всіх акаунтах"
```

**Доступні періоди:** today, yesterday, 3d, 7d, 14d, 30d, custom (будь-які дати)

**Доступні розбивки:**
- По віку (age)
- По статі (gender)
- По країні (country)
- По пристрою (device)
- По платформі (publisher_platform — Facebook vs Instagram)

**Що побачиш:**
- Health Score кожного adset з розбивкою по компонентах
- CPL vs план (з % відхилення)
- Тренди: 3d → 7d (стає краще чи гірше?)
- Топ найкращих і найгірших оголошень
- Аналіз аудиторій

### Оптимізація

#### `/ads-optimizer` — Оптимізатор

**Для чого:** Автоматичний аналіз ефективності + план дій з бюджетами.

**Як використовувати:**
```
"Оптимізуй Bas Dent"
"Проведи оптимізацію по всіх акаунтах"
/ads-optimizer
```

**Що відбувається під капотом:**

1. **Збір даних** — метрики за 5 періодів (today, yesterday, 3d, 7d, 30d)
2. **Health Score** — рахує оцінку для кожного adset:
   - CPL Gap (наскільки далеко від плану) — вага 45%
   - Тренди (3d vs 7d — стає краще?) — вага 15%
   - Діагностика (CTR, CPM, частота) — до -30 штраф
   - Новизна (менше 48 годин — м'якша оцінка)
   - Об'єм даних (мало даних = менше довіри)
3. **Ad-Eater Detection** — знаходить "пожирачів бюджету"
4. **План дій** — для кожного adset: збільшити / зменшити / пауза / не чіпати
5. **Нові adsets** — пропонує створити з невикористаних креативів
6. **Підтвердження** — чекає ваш "ОК"
7. **Виконання** — змінює бюджети, ставить на паузу
8. **Лог** — записує все в `history/`

**Приклад результату:**

```
📊 Оптимізація: Bas Dent (act_805414428109857)

AdSet                    HS    Клас         CPL   План   Дія
─────────────────────────────────────────────────────────────
implants_25-35_almaty   +42   very_good    $3.20  $4    Budget $15 → $20 (+33%)
braces_lookalike         +8   good         $4.50  $4    Тримати
whitening_interests      -3   neutral      $5.10  $4    Моніторинг
cleaning_retarget       -38   bad          $12.00 $4    ⛔ ПАУЗА

🐛 Ad-Eaters:
  cleaning_retarget: CPL $12 = 3x від плану → CRITICAL

💡 Нові adsets (невикористані креативи):
  Креатив "veneers_v2" → можна запустити в implants_25-35_almaty

Підтвердити? (так/ні)
```

#### `/creative-analyzer` — Аналіз креативів

**Для чого:** Зрозуміти які креативи (відео, картинки) працюють, а які ні.

**Як використовувати:**
```
"Проаналізуй креативи по Bas Dent"
"Які креативи найкращі за тиждень?"
/creative-analyzer
```

**Що побачиш:**
- **Risk Score** (0-100) для кожного креативу:
  - 0-25 🟢 Low Risk — масштабуй
  - 26-50 🟡 Medium — спостерігай
  - 51-75 🔴 High — оптимізуй
  - 76-100 ⛔ Critical — вимикай
- Групування по creative_tag (тег береться з назви оголошення)
- A/B порівняння між креативами
- Ознаки "втоми креативу" (creative fatigue)
- Рекомендації: що замінити, що масштабувати

**Як працює групування:**
```
Оголошення: kitchen_30-45_almaty → creative_tag = "kitchen"
Оголошення: kitchen_lookalike    → creative_tag = "kitchen"
Оголошення: clinic_25-35_nur     → creative_tag = "clinic"

Результат: всі "kitchen" об'єднуються → один Risk Score
```

### Створення кампаній

#### `/campaign-manager` — Менеджер кампаній

**Для чого:** Створити нову кампанію з нуля: Campaign → AdSet → Ad.

**Як використовувати:**
```
"Створи нову кампанію для Bas Dent, напрям implants, бюджет $20"
"Додай новий adset в існуючу кампанію"
"Створи кампанію на ліди для Smile Clinic"
/campaign-manager
```

**Що потрібно для створення:**
- Акаунт (бриф має бути готовий)
- Об'єктив (LEADS, TRAFFIC, AWARENESS, SALES)
- Таргетинг (вік, стать, гео, інтереси)
- Креатив (відео або картинка)
- Бюджет
- Назва (за правилами naming convention)

**Структура Facebook Ads:**
```
Campaign (Кампанія)
  └── AdSet (Група оголошень) — бюджет, аудиторія, розклад
        └── Ad (Оголошення) — креатив, текст, посилання
```

**Типи бюджетів:**
- **CBO** (Campaign Budget Optimization) — бюджет на рівні кампанії, Facebook сам розподіляє
- **ABO** (Ad Set Budget Optimization) — бюджет на кожну групу окремо, ви контролюєте

---

#### `/targeting-expert` — Таргетинг

**Для чого:** Знайти правильну аудиторію для реклами.

**Як використовувати:**
```
"Знайди інтереси для стоматології"
"Покажи гео-локації Алмати"
"Створи Lookalike аудиторію з покупців"
"Оціни розмір аудиторії: жінки 25-45, Алмати, стоматологія"
/targeting-expert
```

**Можливості:**
- **Пошук інтересів** — повертає ID, назву, розмір аудиторії
- **Пошук гео** — міста, регіони, країни з ключами для API
- **Lookalike** — створення аудиторій схожих людей (1%, 3%, 5%, 10%)
- **Оцінка розміру** — скільки людей у вашій аудиторії
- **Демографія** — вік, стать, мова, освіта

**Lookalike — як працює:**
```
Ваші покупці (seed audience)
        ↓
Facebook аналізує їх профілі
        ↓
Знаходить схожих людей
        ↓
1% = найбільш схожі (маленька, якісна аудиторія)
10% = менш схожі (велика, дешевша аудиторія)
```

### Креативи

#### `/creative-copywriter` — Копірайтер

**Для чого:** Написати текст для реклами. 6 форматів.

**Як використовувати:**
```
"Напиши storytelling для Bas Dent, напрям implants"
"Створи прямий оффер для відбілювання зубів"
"Напиши пост для Telegram каналу"
"Адаптуй цей текст конкурента під наш бізнес"
/creative-copywriter
```

**6 форматів тексту:**

| Формат | Для чого | Приклад початку |
|--------|----------|-----------------|
| **STORYTELLING** | Довга історія для FB/IG | "Мені було 35, коли я втратив зуб..." |
| **DIRECT_OFFER** | Короткий оффер | "Імплант за 1 день. Без болю. $500." |
| **EXPERT_VIDEO** | Скрипт для лікаря/експерта | "Чому імпланти — це не страшно?" |
| **TELEGRAM_POST** | Пост для каналу | Інформаційний, без прямого продажу |
| **THREADS_POST** | Короткий пост | Провокаційне питання/тезис |
| **REFERENCE** | Адаптація чужого тексту | Беремо структуру конкурента |

**4 компоненти** (можна генерувати окремо):
- **OFFER** — заголовок (6-12 слів)
- **BULLETS** — 3 переваги (по 5 слів)
- **PROFITS** — 1 бонус/акція
- **CTA** — заклик до дії (2-5 слів)

## Health Score

### Формула

```
Health Score = (CPL_Gap + Trends + Diagnostics + Today_Adj) × Volume_Factor

Де:
  CPL_Gap (вага 45):     Наскільки CPL далеко від плану
  Trends (вага 15):      3d CPL vs 7d CPL (тренд)
  Diagnostics (до -30):  Штрафи за низький CTR, високий CPM, частоту
  Today_Adj (0 до +30):  Бонус якщо сьогодні краще за вчора
  Volume_Factor (0.6-1):  Множник довіри (мало даних = менше довіри)
```

### Приклад розрахунку

```
AdSet: implants_25-35_almaty
  План CPL: $4
  Факт CPL (7d): $3.20 (на 20% дешевше від плану)

  CPL_Gap:    +30  (20% дешевше → позитивний)
  Trends:     +10  (3d CPL $3.00 < 7d CPL $3.20 → покращується)
  Diagnostics: -8  (CTR 0.9% < 1% → штраф)
  Today_Adj:   +5  (сьогодні CPL $2.80 < вчора $3.50)
  Volume:      0.9 (4200 impressions)

  HS = (30 + 10 - 8 + 5) × 0.9 = 33.3 → +33
  Клас: very_good → масштабувати +20-30%
```

## Правила безпеки

### Небезпечні операції (завжди запитують підтвердження)

- Зміна бюджетів (збільшення або зменшення)
- Пауза або відновлення кампаній, adsets, ads
- Створення нових кампаній
- Видалення чого-небудь
- Все що впливає на витрати

### Ліміти

| Правило | Значення |
|---------|----------|
| Макс. збільшення бюджету | +30% за одну дію |
| Макс. зменшення бюджету | -50% за одну дію |
| Мін. бюджет adset | $3/день |
| Рекомендований макс. бюджет | $100/день |
| Не створювати adsets після | 18:00 (за часовим поясом акаунту) |
| Мін. дані для рішення | 1000 імпресій, 3 конверсії, 2 дні |
| Learning phase | ~7 днів — не чіпати нові adsets |
| Макс. зменшень поспіль | 3 за 3 дні → пауза замість зменшення |

### Логування

Кожна дія записується в `history/YYYY-MM/YYYY-MM-DD.md`:

```
## 2026-03-02

### 09:15 — Оптимізація Bas Dent
- implants_25-35_almaty: бюджет $15 → $20 (+33%)
- cleaning_retarget: ПАУЗА (ad-eater, CPL $12 = 3x від плану)
- Причина: Health Score аналіз
```

## Поради та лайфхаки

### 1. Використовуйте природну мову

Не обов'язково знати точні команди. Просто опишіть що хочете:
- "Як справи з рекламою?" → dashboard
- "Що не так з Bas Dent?" → reporter + analyzer
- "Створи щось нове для Instagram" → copywriter + image generator

### 2. Бриф — це головне

Бриф (`config/briefs/{account}.md`) — найважливіший файл. Від нього залежить:
- Яку CPL агент вважає нормальною
- Які бюджети рекомендує
- Які аудиторії використовує
- Які креативи пропонує

**Оновлюйте бриф** коли змінюються бізнес-цілі.

### 3. Правильні назви = правильна аналітика

Називайте оголошення за конвенцією: `creative_tag_аудиторія_гео`

```
✅ Добре:  kitchen_25-45_almaty     → tag "kitchen"
✅ Добре:  promo_lookalike_nur      → tag "promo"
❌ Погано: новое_объявление_1       → tag "новое" (не інформативно)
❌ Погано: ad_copy_v3_final_FINAL   → tag "ad" (занадто загально)
```

### 4. Не оптимізуйте занадто часто

- Нові adsets потребують 5-7 днів на "навчання"
- Часті зміни бюджету скидають алгоритм Facebook
- Оптимально: 2-3 рази на тиждень

### 5. Дивіться на тренди, не на один день

Поганий день ≠ поганий adset. Дивіться на 7d:
- Один день CPL $8 при плані $4 — може бути випадковість
- 7 днів CPL $8 при плані $4 — треба діяти

### 6. Не ігноруйте Ad-Eaters

Якщо агент знайшов "пожирача бюджету" рівня Critical (CPL > 3x від плану) — вимикайте одразу. Кожна година = зайві витрати.

### 7. Реєструйте креативи

Додавайте нові креативи в `config/creatives.md` — тоді `/ads-optimizer` зможе пропонувати нові adsets з невикористаними креативами.

### 8. Перевіряйте логи

`history/` — ваша "бухгалтерія змін". Якщо щось пішло не так — завжди можна подивитись що і коли змінювалось.
