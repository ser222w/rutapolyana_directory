# Гід для CEO — AI Ads Agent

> Цей документ пояснює простими словами, що робить AI Ads Agent, як ним користуватись, і що ви бачитимете як керівник компанії.

---

## Що це таке?

AI Ads Agent — це ваш **штучний інтелект-маркетолог**, який живе в терміналі (Claude Code). Він вміє:

- Дивитись на всю рекламу по всіх акаунтах одним поглядом
- Показувати зрозумілі звіти: скільки витратили, скільки лідів, яка ціна ліда
- Знаходити "пожирачів бюджету" — оголошення які їдять гроші, але не дають результат
- Оптимізувати бюджети — додавати гроші тому що працює, забирати у того що не працює

**Аналогія:** Уявіть, що у вас є аналітик, який 24/7 дивиться на рекламний кабінет і каже: "Ось це працює — додаємо бюджет. Ось це зливає гроші — вимикаємо."

## Ваші 4 основні команди

Вам як CEO потрібні лише 4 команди. Все решта — для маркетолога.

### 1. `/dashboard` — Загальна картина

**Що робить:** Показує таблицю по ВСІМ акаунтам одразу.

**Коли використовувати:** Щоранку або коли хочете зрозуміти "як справи з рекламою".

**Що ви побачите:**

```
┌──────────────┬──────────┬───────┬────────┬──────────┐
│ Акаунт       │ Витрати  │ Ліди  │ CPL    │ План     │
├──────────────┼──────────┼───────┼────────┼──────────┤
│ Bas Dent     │ $280     │ 65    │ $4.31  │ $4 ✅    │
│ Smile Clinic │ $150     │ 22    │ $6.82  │ $5 ⚠️    │
│ AutoParts    │ $520     │ 180   │ $2.89  │ $3 ✅    │
└──────────────┴──────────┴───────┴────────┴──────────┘
```

- **CPL** (Cost Per Lead) — скільки коштує один лід (заявка)
- **План** — яку ціну ми хочемо. ✅ = в нормі, ⚠️ = дорожче за план

**Просто напишіть:** `/dashboard` і все.

---

### 2. `/ads-reporter` — Детальний звіт

**Що робить:** Показує детальний звіт за обраний період.

**Коли використовувати:** Щотижня на планерці, або коли хочете розібратись в деталях.

**Що можна запитати:**
- "Покажи звіт по Bas Dent за тиждень"
- "Порівняй цей тиждень з минулим"
- "Що з рекламою за березень?"

**Що ви побачите:**
- Витрати, ліди, CPL за кожну кампанію
- Тренди: стає дорожче чи дешевше?
- Найкращі та найгірші оголошення
- Аудиторія: яка вікова група купує найкраще

---

### 3. `/ads-optimizer` — Оптимізація (з підтвердженням)

**Що робить:** Аналізує ВСЮ рекламу, знаходить проблеми, пропонує план дій.

**Коли використовувати:** 2-3 рази на тиждень, або делегуйте маркетологу.

**Як це працює:**

1. Агент сканує всі оголошення
2. Рахує **Health Score** (оцінка здоров'я) для кожного: від -100 до +100
3. Знаходить "пожирачів бюджету" (ad-eaters)
4. Показує план: "Пропоную зробити ось це..."
5. **Чекає вашого "ОК"** — нічого не змінює без дозволу!

**Приклад плану:**

```
🟢 implants_25-35_almaty — HS: +42 (відмінно) → Збільшити бюджет $15 → $20
🟡 braces_lookalike      — HS: +3  (нейтрально) → Не чіпати
🔴 whitening_all_ages     — HS: -35 (погано) → Зменшити $20 → $10
⛔ cleaning_retarget      — HS: -60 (пожирач) → Поставити на паузу
```

Ви відповідаєте "так" або "ні" — і агент виконує.

---

### 4. Запитати щось своїми словами

Агент розуміє природну мову. Просто запитайте:

- "Скільки ми витратили на рекламу за лютий?"
- "Який акаунт найприбутковіший?"
- "Чому Smile Clinic такий дорогий?"
- "Покажи топ-5 найкращих оголошень"

## Що ВАЖЛИВО знати

### Безпека ваших грошей

| Правило | Що це значить |
|---------|---------------|
| Підтвердження | Агент НІКОЛИ не змінює бюджет без вашого "ОК" |
| Ліміт +30% | Бюджет можна збільшити максимум на 30% за раз |
| Ліміт -50% | Бюджет можна зменшити максимум на 50% за раз |
| Мін. бюджет | Не менше $3 на одну групу оголошень |
| Макс. бюджет | Рекомендовано не більше $100 на одну групу |
| Історія | Все записується в лог — можна перевірити що і коли змінювалось |

### Health Score — просте пояснення

Кожне оголошення отримує оцінку від -100 до +100:

| Оцінка | Що це | Що робимо |
|--------|-------|-----------|
| +25 і вище | Відмінно | Даємо більше грошей (+20-30%) |
| +5 до +24 | Добре | Тримаємо як є |
| -5 до +4 | Нейтрально | Спостерігаємо |
| -6 до -25 | Погано | Зменшуємо бюджет |
| -25 і нижче | Дуже погано | Вимикаємо або різко зменшуємо |

### Що таке "пожирач бюджету" (Ad-Eater)?

Оголошення, яке коштує набагато дорожче за план:

- **Критичний** (CPL > 3x від плану) — Наприклад, план $5, а це оголошення коштує $16. Рекомендація: вимкнути.
- **Високий** (CPL > 2x) — План $5, коштує $11. Рекомендація: сильно зменшити бюджет.
- **Середній** (CPL > 1.5x) — План $5, коштує $8. Рекомендація: зменшити трохи.

## Делегування маркетологу

Ви можете повністю делегувати оптимізацію маркетологу. Тоді ваш ритуал ще простіший:

1. **Раз на день:** `/dashboard` — 30 секунд подивитись чи все ОК
2. **Раз на тиждень:** попросити маркетолога показати звіт `/ads-reporter`
3. **Раз на місяць:** детальний розбір ефективності

Маркетолог буде використовувати `/ads-optimizer` та інші команди щодня — і ви побачите результат на дашборді.

## Словник термінів

| Термін | Простими словами |
|--------|-----------------|
| **CPL** | Ціна одного ліда (заявки). Чим менше — тим краще |
| **CTR** | % людей що клікнули на рекламу. Норма: 1-3% |
| **CPM** | Ціна за 1000 показів реклами |
| **Health Score** | Оцінка здоров'я оголошення (-100 до +100) |
| **Ad-Eater** | Оголошення що "їсть" бюджет без результату |
| **AdSet** | Група оголошень з однаковою аудиторією |
| **Campaign** | Кампанія — містить одну або кілька груп (AdSets) |
| **Lookalike** | Аудиторія схожих людей (Facebook шукає людей схожих на ваших клієнтів) |
| **ROAS** | Повернення на рекламні витрати. ROAS 3 = на кожен $1 заробив $3 |
| **Бриф** | Документ з налаштуваннями: ціна ліда, бюджет, аудиторія |
